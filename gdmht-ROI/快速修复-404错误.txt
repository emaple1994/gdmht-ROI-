========================================
Netlify 404 错误 - 快速修复指南
========================================

【问题】看到 "Page not found" 错误

【原因】Netlify 没有正确识别 index.html 文件

【解决方案】

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

✅ 方法一：重新部署（最简单，推荐）

步骤：
1. 确保以下文件都在同一文件夹中：
   ✓ index.html
   ✓ netlify.toml（我已创建）
   ✓ _redirects（我已创建）

2. 访问 Netlify 拖拽页面：
   https://app.netlify.com/drop

3. 将包含所有文件的文件夹拖拽到页面
   （或者选中所有三个文件后拖拽）

4. 等待部署完成（约30秒）

5. 点击网站链接，应该就能正常访问了！

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

✅ 方法二：通过 Netlify 控制台重新部署

步骤：
1. 登录 Netlify 控制台：
   https://app.netlify.com

2. 找到您的网站，点击进入

3. 进入 "Site settings" → "Deploys"

4. 点击 "Trigger deploy" → "Deploy site"

5. 上传包含以下文件的文件夹：
   - index.html
   - netlify.toml
   - _redirects

6. 等待部署完成

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

✅ 方法三：检查现有部署

如果网站已经部署过：

1. 登录 Netlify 控制台
2. 进入您的网站
3. 点击 "Deploys" 标签
4. 查看最新的部署
5. 点击部署详情
6. 检查文件列表中是否有 index.html

如果没有 index.html：
→ 需要重新上传包含 index.html 的文件夹

如果有 index.html 但仍然404：
→ 尝试清除缓存：
   Site settings → Build & deploy → Clear cache and deploy site

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

📋 必需文件清单

确保以下文件都在部署文件夹中：

☐ index.html          （主文件，必需）
☐ netlify.toml        （配置文件，我已创建）
☐ _redirects          （重定向文件，我已创建）

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

🚀 最快解决方案（3步）

1. 选中以下三个文件：
   - index.html
   - netlify.toml
   - _redirects

2. 访问并拖拽：
   https://app.netlify.com/drop

3. 完成！等待部署，然后访问网站。

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

💡 替代方案：使用 Vercel

如果 Netlify 持续出现问题，可以使用 Vercel：

1. 访问：https://vercel.com
2. 注册/登录
3. 点击 "Add New Project"
4. 上传包含 index.html 的文件夹
5. 点击 "Deploy"
6. 完成！
